## Watching the Process Unfold

The assistant got straight to work, searching for high-resolution Earth textures and considering the best tools for interactive 3D visualization. When the user suggested “three.js,” the assistant didn’t miss a beat. It shifted gears and began scaffolding the project with Three.js, a powerful JavaScript library for 3D graphics in the browser.

I watched as it:

- **Set up the development environment** (using tools like Vite for fast prototyping)
- **Initialized a Three.js scene, camera, and renderer**
- **Created the Earth sphere** with `THREE.SphereGeometry`
- **Fetched and applied realistic Earth textures**
- **Added dynamic lighting and mouse controls** for a smooth, interactive experience

## The Code That Brought Earth to Life

Here’s a glimpse of the core code the assistant generated:

```jsx
import * as THREE from 'three';

*// Scene setup*
const scene = new THREE.Scene();
const camera = new THREE.PerspectiveCamera(75, window.innerWidth/window.innerHeight, 0.1, 1000);
camera.position.z = 2;

*// Renderer*
const renderer = new THREE.WebGLRenderer();
renderer.setSize(window.innerWidth, window.innerHeight);
document.body.appendChild(renderer.domElement);

*// Earth geometry and material*
const geometry = new THREE.SphereGeometry(0.5, 32, 32);
const material = new THREE.MeshPhongMaterial({
  map: new THREE.TextureLoader().load('textures/earthmap.jpg'),
  bumpMap: new THREE.TextureLoader().load('textures/earthbump.jpg'),
  bumpScale: 0.05,
});
const earth = new THREE.Mesh(geometry, material);
scene.add(earth);

*// Lighting*
const light = new THREE.PointLight(0xffffff, 1, 100);
light.position.set(5, 3, 5);
scene.add(light);

*// Animation loop*
function animate() {
  requestAnimationFrame(animate);
  earth.rotation.y += 0.002;
  renderer.render(scene, camera);
}
animate();
```
